Hexo使用

_config.yml

可以在_config.yml里修改大部分配置文件

网站

  • title:网站标题
  • subtitile:网站副标题
  • description:网站描述
  • keywords:网站的关键词,支持多个关键词
  • author:你的名字
  • language:网站使用的语言
  • timezone:时区,一般用的是Asia或者Shanghai

其中descrption主要用于seo,告诉搜索引擎一个简单的描述,author用于显示主题文章的作者

网址

  • url 网址,必须以http或者https开头
  • root 网站根目录
  • permalink 文章的永久连接格式
  • permalink_defaults 永久链接中各部分的默认值
  • pretty_urls.strailing_index 是否在永久链接中保留尾部的index.html 设置为false是去除,默认值为true
  • pretty_urls.trailing_html 是否在永久链接中保留尾部的.html 设置为false是去除,对index.html无效,默认值为true

目录

  • source_dir 资源文件夹,这个文件夹来存放内容
  • public_dir 公共文件夹,这个文件夹用于存放生成的站点文件
  • tag_dir 标签文件夹
  • archive_dir 归档文件夹
  • category_dir 分类文件夹
  • code_dir Include code文件夹,source_dir下的子目录
  • i18n_dir 国际化(i18n)文件夹
  • skip_render 跳过指定文件的渲染,匹配到文件将会不做改动的复制到 public 目录中,可以使用glob表达式来匹配路径

对于hexo新手来说通常不建议动这一部分的值

文章

  • new_post_name 新文章的文件名称
  • default_layout 预设布局
  • auto_spacing 在中文英文之间加入空格
  • titlecase 把标题转换为title case
  • external_link 在新标签页打开链接
  • external_link.enable 在新标签页打开链接
  • external_link.field 对阵个网站(site)生效或者仅对文章(post)生效
  • external_link.exclude 需要排除的域名,主域名和子域名如www需要分别配置
  • filename_case 把文件名称转换为(1)小写或者(2)大写
  • render_drafts 显示草稿
  • post_asset_folder 启动Asset文件夹
  • relative_link 把链接改为与根目录的相对位置
  • future 显示未来的文章
  • highlight 代码块的设置,参考Highlight.js进行设置
  • prismjs 代码块的设置,参考PrismJS进行设置

分类 &标签

  • default_category 默认分类
  • category_map 分类别名
  • tag_map 标签别名

日期/时间格式

Hexo用Moment.js来解析和显示时间

  • date_format 日期格式
  • time_format 时间格式
  • updated_option 当在Front Matter中没有指定updated时updated的取值

updated_option
updated_option 控制了当Front Matter中没有指定时updated如何取值:

  • mtime:使用文件的最后修改时间,是默认的设置
  • date:使用date作为update的值.可以被用于Git工作流之中,因为使用Git管理站点时文件的追后秀爱日期常常会被改变
  • empty:直接删除updated,使用这个选项可能会让大部分主题和插件无法工作

分页

  • per_page 每页显示的文章量,0关闭分页功能
  • pagination_dir 分页目录

扩展

  • theme 当前主题名称为false时禁用主题
  • theme_config 主题的配置文件放在这里会覆盖主题目录下的 _config.yml 中的配置
  • deploy 部署部分的设置
  • meta_generator Meta generator标签,值为false时Hexo不会在头部插入该标签

包括或不包括目录和文件

在Hexo配置文件中,通过设置include/exclude可以让Hexo进行处理或忽略某些目录和文件夹,可以使用golb表达式对目录文件进行匹配

include和exclude选项只会用于source 而ignore 选项会应用到所有文件夹

  • include Hexo默认会不包括source/下的文件和文件夹(包括名称以_和.开头的文件和文件夹, Hexo的_posts和_data等目录除外)通过设置此字段使Hexo处理他们并将他们复制到source目录下
  • exclude Hexo不包括source/下的这些文件和目录
  • ignore Hexo会忽略整个Hexo项目下的这些文件夹或文件

指令

hexo init [folder]

新建一个网站,如果没有设置folderHexo在目前默认文件夹建立网站

hexo new [layout] <title>

新建一篇文章.如果没有设置layout的话,默认使用 _config.yml中的default_layout参数来代替,如果包含空格的话使用引号括起来

  • -p,--path 自定义新文章的路径
  • -r,--replace 如果存在同名文章,将其替换
  • -s,--slug 文章的Slug,作为新文章的文件名和发布后的URL

默认情况下Hexo会使用文章的标题来决定文章的路径.对于独立页面来说,Hexo会创建一个以标题为名字的目录,并在目录中放置一个index.md文件.可以使用--path参数来覆盖上述行为,自行决定文件目录

注意!!! title必须是指定的

hexo generate

生成静态文件

  • -d,--deploy 文件生成后立即部署网站
  • -w,--watch 监视文件变动
  • -b,--bail 生成过程中如果发生任何未处理的异常则抛出异常
  • -f,--force 强制重新生成文件Hexo引入了差分机制,如果public目录存在,那么hexo g指挥重新生成改动的文件 使用该参数的效果接近hexo clean && hexo generate
  • -c,--concurrency 最大同时生成文件的数量,默认无限制

发表草稿

hexo server

启动服务器,默认情况下,访问网址为:http://localhost:4000/

  • -p,--port 重设端口
  • -s,--static 只使用静态文件
  • -l,--log 启动日记记录,使用覆盖记录格式

部署网站

  • -g,--generate 部署之前预先生成的静态文件
